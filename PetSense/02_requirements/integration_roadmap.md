# 事業計画書とmockアプリの整合性向上ロードマップ

## エグゼクティブサマリー

PetSenseの改良版事業計画書と mock アプリの設計には、**7つの主要なギャップ**が存在します。これらを段階的に解決することで、mock アプリは事業計画書の戦略的方向性と完全に整合し、投資家へのプレゼンテーション資料としても機能するようになります。

本ロードマップは、**短期（1-3ヶ月）、中期（4-6ヶ月）、長期（7-12ヶ月）**の3段階で、優先度の高い順に改善を進める計画です。

---

## 1. 全体構成図

```
事業計画書（フェーズ1-3）
    ↓
ギャップ分析（7つの主要ギャップを特定）
    ↓
優先度マトリックス（事業価値 × 実装難度）
    ↓
段階的ロードマップ（短期・中期・長期）
    ↓
各段階での実装詳細と工数見積もり
```

| 優先度 | ギャップ | 事業価値 | 実装難度 | 工数 | 対象フェーズ | 理由 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **1位** | 尿分析 UI | 高 | 中 | 3-4週 | フェーズ1 | MVP の最重要機能。ユーザー価値の中核。 |

## 3. 短期ロードマップ（1-3ヶ月）: MVP の完成

### 目標
フェーズ1 の MVP を完成させ、尿分析と異常検知による基本的なペット健康管理機能を実装する。

### 実装内容

#### 3.1 尿分析 UI の実装（優先度1位）

**実装対象**: 記録画面に「計測追加」機能を統合

**追加機能**:

1. **計測記録入力フォーム**
   - 計測日時の選択
   - 計測タイプの選択（尿 / フン / 体重）
   - 尿の計測項目（pH値、比重、タンパク質、ケトン体、白血球、赤血球）
   - フンの計測項目（ブリストル・スケール、色、形状）
   - 体重の入力

2. **計測結果表示画面**
   - 最新の計測値を大きく表示
   - 正常範囲との比較（色分け: 正常=緑、異常=赤）
   - 時系列グラフ（1週間、1ヶ月、3ヶ月の切り替え可能）

3. **データベース拡張**
   - `measurements` テーブルに詳細項目を追加
   - `normal_ranges` テーブルを新規作成（正常範囲の定義）

**実装工数**: 3-4週間

**技術スタック**: React Native + TypeScript + expo-sqlite

**成果物**: 
- 計測記録入力画面コンポーネント
- 計測結果表示コンポーネント
- グラフ表示ライブラリの統合（react-native-chart-kit など）
- DB スキーマの拡張

---

#### 3.2 アラート判定ロジックの実装（優先度2位）

**実装対象**: ホーム画面にアラート表示、アラート画面を本格実装

**追加機能**:

1. **異常判定ルールの定義**
   - 尿のpH値が 6.0-7.0 の正常範囲外 → 「尿pH異常」
   - タンパク質が陽性 → 「タンパク尿」
   - ケトン体が陽性 → 「ケトン体陽性」
   - ブリストル・スケールが 1 or 7 → 「便秘/下痢」
   - 体重が1週間で 10% 以上変化 → 「体重急変」
   - 排泄回数が通常の ±30% 以上変化 → 「排泄異常」

2. **アラート表示**
   - ホーム画面に「異常検知」バナーを表示
   - アラート画面で詳細情報（異常内容、発生日時、推奨アクション）を表示
   - 獣医師への相談提案ボタン

3. **通知機能**
   - ローカル通知（アプリ内トースト）
   - 重要度に応じた通知レベル（警告 / 注意 / 情報）

**実装工数**: 2-3週間

**技術スタック**: React Native + TypeScript + ローカル通知 API

**成果物**:
- アラート判定エンジン
- アラート表示コンポーネント
- 通知ロジック

---

#### 3.3 デバイス統合の基本実装（優先度4位）

**実装対象**: デバイス管理画面の拡張

**追加機能**:

1. **デバイスタイプの明示**
   - `DeviceState` を拡張し、デバイスタイプ（スマートシーツ、スマートベース、給餌器、活動量計）を管理
   - デバイス固有の状態（バッテリー残量、最終同期時刻、センサー状態）を管理

2. **デバイスダッシュボード**
   - 接続状態の可視化（オンライン / オフライン）
   - バッテリー残量の表示（%）
   - 最終同期時刻の表示
   - デバイス固有の情報（センサー状態など）

3. **デバイス登録フロー**
   - デバイスタイプの選択
   - ペアリング（Bluetooth / Wi-Fi）
   - ペットへの紐付け

**実装工数**: 3-4週間

**技術スタック**: React Native + TypeScript + Bluetooth/Wi-Fi API

**成果物**:
- 拡張された `DeviceState` 型定義
- デバイスダッシュボード画面
- デバイス登録フロー

## 4. 中期ロードマップ（4-6ヶ月）: フェーズ2 への準備

### 目標
給餌器連動ロジックを実装し、フェーズ2 の核となる機能を完成させる。

### 実装内容

#### 4.1 給餌器連動ロジックの実装（優先度3位）

**実装対象**: 食事画面の大幅拡張、給餌器管理画面の新規作成

**追加機能**:

1. **ペット情報の拡張**
   - 犬種（ドロップダウンから選択）
   - 体重（kg）
   - 年齢（月齢）
   - 誕生日
   - 活動レベル（低 / 中 / 高）
   - アレルギー情報

2. **給餌器管理**
   - 給餌器デバイスの登録（デバイス統合と連携）
   - 給餌スケジュール設定（朝・昼・晩など、複数設定可能）
   - 給餌量の設定（g / kcal）

3. **カロリー計算エンジン**
   - 犬種別の栄養基準データベース（例: ラブラドール・レトリバーの成犬は1日 1,200-1,500 kcal）
   - 個体の体重に基づいた適正カロリー計算
   - 体重・尿データに基づいた給餌量の動的調整
   - 専用フード（複数ブランド）のカロリー/100g データベース

4. **給餌器への指令送信**
   - 計算された給餌量をデバイスに送信（シミュレーション）
   - 給餌実績の記録（実際に給餌されたかどうか）

5. **栄養管理ダッシュボード**
   - 本日の適正カロリー表示
   - 本日の摂取カロリー表示
   - 栄養バランスの評価（タンパク質、脂肪、炭水化物）
   - 推奨フードの表示

**実装工数**: 5-6週間

**技術スタック**: React Native + TypeScript + バックエンド API（カロリー計算）

**成果物**:
- ペット情報拡張フォーム
- 給餌器管理画面
- カロリー計算エンジン
- 栄養管理ダッシュボード

---

#### 4.2 レポート出力の実装（優先度5位）

**実装対象**: レポート画面の本格実装

**追加機能**:

1. **週次レポート**
   - 体重の推移グラフ
   - 排泄回数・状態の集計（表とグラフ）
   - 食事摂取量の集計
   - 異常検知の記録（発生日時、内容）

2. **月次レポート**
   - 健康指標の月間サマリー（体重、排泄、食事）
   - 前月比の変化（%）
   - トレンド分析（改善 / 悪化 / 変化なし）

3. **獣医師向けレポート**
   - 医学的に整理されたデータ形式（尿検査結果、便の状態など）
   - PDF 出力機能
   - 共有可能なリンク生成（一時的な有効期限付き）

4. **共有可能なレポート**
   - 家族メンバーに共有できるシンプル版
   - SNS 共有機能（LINE、メールなど）

**実装工数**: 3-4週間

**技術スタック**: React Native + TypeScript + PDF ライブラリ（react-native-pdf-lib など）

**成果物**:
- 週次・月次レポート画面
- PDF 出力機能
- 共有機能

#### 4.3 家族共有機能の基本実装（優先度6位）

**実装対象**: 設定画面に「家族共有」セクションを追加

**追加機能**:

1. **招待機能**
   - QRコード生成・読み込み（ペアリング用）

   - 招待リンク生成（一時的な有効期限付き）

2. **権限管理**
   - 閲覧のみ
   - 編集可能（記録追加・編集）
   - 管理者権限（ユーザー管理、デバイス管理）

3. **リアルタイム同期の準備**
   - ローカルDB での競合解決ロジックの実装
   - バックエンド API への同期準備（実装は後期）

**実装工数**: 4-5週間（バックエンド API が必要な場合は追加工数）

**技術スタック**: React Native + TypeScript + バックエンド API

**成果物**:
- 家族共有管理画面
- 招待機能
- 権限管理ロジック

## 5. 長期ロードマップ（7-12ヶ月）: フェーズ3 への準備

### 目標
活動量計連動、B2B 機能を実装し、フェーズ3 の完全ヘルスケアエコシステムへの基盤を構築する。

### 実装内容

#### 5.1 活動量計連動の実装（優先度4位の拡張）

**実装対象**: デバイス統合の拡張、ホーム画面に活動量表示を追加

**追加機能**:

1. **活動量計デバイスの統合**
   - 首輪型活動量計との Bluetooth ペアリング
   - 運動量、心拍数、睡眠時間の記録

2. **活動量ダッシュボード**
   - 本日の運動量（ステップ数、距離、消費カロリー）
   - 心拍数の推移グラフ
   - 睡眠時間の表示

3. **食事・排泄・活動の統合分析**
   - 「運動量と食事摂取量の相関」を分析

   - 推奨カロリーの動的調整（運動量に基づいて）

**実装工数**: 4-5週間

**追加機能**:

   - 予防医療への保険給付の申請

2. **保険ダッシュボード**
   - 現在の割引率表示

**実装工数**: 4-5週間（バックエンド API が必要）

#### 5.4 AI ヘルスコーチの実装（優先度7位の一部）

   - 蓄積されたペットの健康データから、最適なプランを学習

**実装工数**: 6-8週間（機械学習モデルの構築が必要）

## 7. 各段階での投資対効果（ROI）

### 短期フェーズ完了後
- **実装工数**: 9-12週間
- **達成機能**: 尿分析、アラート判定、デバイス統合（基本）
- **ユーザー価値**: MVP として機能。基本的なペット健康管理が可能。
- **ビジネス価値**: フェーズ1 の機能が完成。初期ユーザー獲得が可能。

### 中期フェーズ完了後
- **実装工数**: 12-16週間（短期を含めて合計 21-28週間）
- **達成機能**: 給餌器連動、レポート出力、家族共有
- **ユーザー価値**: 月額ARPU が ¥2,000 → ¥6,000 に増加。ユーザー満足度が大幅向上。
- **ビジネス価値**: フェーズ2 の機能が完成。収益が3倍に増加。

### 長期フェーズ完了後
- **実装工数**: 20-24週間（短期+中期を含めて合計 41-52週間）
- **達成機能**: 活動量計連動、獣医師ネットワーク、保険連携、AI ヘルスコーチ
- **ユーザー価値**: 完全なペットヘルスケアエコシステム。月額ARPU が ¥10,000 に達する。
- **ビジネス価値**: フェーズ3 の機能が完成。B2B 展開が可能。年間売上が 960 億円に達する。

## 8. 実装の依存関係と並行実装の可能性

### 依存関係
- **尿分析 UI** → **アラート判定** （尿分析データが必要）
- **デバイス統合** → **給餌器連動** （給餌器デバイスの管理が必要）
- **給餌器連動** → **カロリー計算** （給餌器制御に必要）
- **レポート出力** → **家族共有** （共有対象としてレポートが必要）

### 並行実装の可能性
- **尿分析 UI** と **デバイス統合** は並行実装可能
- **アラート判定** と **レポート出力** は並行実装可能
- **家族共有** と **給餌器連動** は並行実装可能

**工数削減効果**: 並行実装により、全体工数を 10-15% 削減可能

---

## 9. 技術的な推奨事項

### バックエンド API の必要性
以下の機能を実装する場合、バックエンド API が必要になります。

- **家族共有** （複数ユーザーの同期）
- **獣医師ネットワーク** （獣医師側のダッシュボード）
- **保険連携** （保険会社との連携）
- **AI ヘルスコーチ** （機械学習モデルの推論）

**推奨スタック**: Node.js + Express + PostgreSQL + Firebase（認証・リアルタイム同期）

### フロントエンド技術の進化
- **短期**: React Native のみで実装可能
- **中期**: Redux Toolkit などの状態管理ライブラリの導入を検討
- **長期**: React Navigation などの正式なナビゲーションライブラリの導入を検討

## 10. 結論と推奨アクション

### 推奨される実装順序

1. **即座に開始**: 尿分析 UI とアラート判定ロジックの実装（短期フェーズ）
