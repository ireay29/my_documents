### 1. GitHub Copilot 料金改定でトークン削減が急務に——最新4手法まとめ
**Qiita / 新海 正明**

2026年6月1日から GitHub Copilot が従量課金（AI Credits）に移行。Agent mode を多用していたユーザーは実質的な大幅値上げとなっており、トークン削減の実践手法が注目を集めている。記事では以下の4手法を解説している。

| 手法 | 効果 | 向いているシーン |
|---|---|---|
| **Caveman Prompt**（原始人プロンプト） | 出力14〜45%削減 | 長いエージェントセッション、説明・分析生成 |
| **プロンプトキャッシング** | 入力90%割引（Anthropic公式） | 同じシステムプロンプトを繰り返す場面 |
| **モデルルーティング** | 高コストモデルの消費を抑制 | 軽作業はHaiku/Flash、難問のみOpus |
| **サブエージェント分離** | コンテキスト汚染防止 | ファイル探索・調査が多いタスク |

コード補完は現時点では無制限のまま。まず Copilot ダッシュボードで自分の消費パターンを確認してから対策を選ぶのが現実的とのこと。

### 2. AI活用のつもりで「処理コスト付きのゴミ」を増やしていないか
**Zenn / continuity-model**

「コンテキストを全部渡せばAIは賢くなる」という誤解を正面から批判した論考。コンテキストを増やすことは知識を足すことではなく、**処理対象・料金・検証範囲・ノイズ**をすべて増やすことだと指摘する。

特に業務用途では、創作と異なり「AIの飛躍（補完）」が事故になる。古い情報・未確定情報・AI生成物・怒りの短文返信まで、コンテキストに入れた時点で推論コストが発生する。「捨てる設計」こそがコストと品質を同時に改善するという主張は、AIシステム設計の実務に直結する。

🔗 https://zenn.dev/continuitymodel/articles/1a45801a9da65a

### 3. 永続メモリの落とし穴から生還、コンテキスト94%減できた話
**Zenn / のち＠LiveNode AI**

非エンジニアのコンサルタントが Claude Code のセッション引き継ぎ問題に3ヶ月格闘した末、**Obsidian + Noetic Cell構造 + ステップリーディング**で1ノートあたり94%のトークン削減を実現した実践報告。

設計の核心は「保存時にAIが構造化し、読み出し時の負荷を先に潰す（Upstream Structuring）」という発想。全ノートの「主張」だけを集めたCLAIMS_INDEXを起点に、必要なノートだけを段階的に深掘りするステップリーディングが機能している。573ノート規模でも削減率はほぼ変わらないことが実測で確認されており、OSSとして公開済み。

🔗 https://zenn.dev/seabird/articles/40fe96a2cb75c0

**Zenn / かとう ひろし**

日本ソフトウェア科学会の『2030年のAIとアジャイル レポート』への応答論考。レポートが「適応」を前向きな能力獲得として描いているのに対し、著者は「適応の実体は、後回しにしてきた問題のツケの清算だ」と反論する。

特に印象的な論点として、**蒸気機関から電気への移行に数十年かかった歴史的事例**を引き、「技術より器の更新が遅い」という見立てを提示。また、AIを「道具でも仲間でもない第三のもの（お化け）」として外に出して眺めることで、信頼の較正が可能になるという視点は、AIとの向き合い方を考えるうえで示唆に富む。

さらに、**バイブコーディングの熟達を左右するのは新しいスキルではなく「コードを読む力」と「考えを明確な文章にする力」という古い基礎だった**という研究（CHI 2026）の引用も興味深い。
