## 🤖 AI・機械学習

### AIエージェントのPRが太る前に止める: Ponytailで「書かない判断」をレビュー基準にする
**Zenn** | [記事リンク](https://zenn.dev/53able/articles/66616005d7aaf4)

AIコーディングエージェント（Claude Codeなど）は実装の初速を上げますが、同時に「標準ライブラリで足りる処理に新しい依存関係を入れる」「将来の拡張性として使わない設定項目を持ち込む」といった過剰なコード（太ったPR）を生成しがちです。
この記事では、AI生成コードの肥大化を防ぐためのスキル「Ponytail」を紹介しています。Ponytailは「短いコードを書く」ことではなく、「所有するコード（依存関係や保守対象）を減らす」ことを目的としており、以下の6段階のラダーでAIに「書かない判断」を促します。

1. そもそも必要か（投機的な要求なら作らない）
2. 標準ライブラリで足りるか
3. ネイティブ機能（ブラウザやDB制約など）で足りるか
4. 既存依存で足りるか
5. 1行で済むか
6. それでも無理なら、動く最小実装を書く

AI時代においては、より多く生成する能力ではなく、受け取る差分を小さく保つ能力が保守性を左右するという、非常に重要な視点が提示されています。

## 🧠 心理学・メンタルヘルス

### 「課題の分離って、冷たくないですか?」――そう言われるたびに、私は思う
**note** | [記事リンク](https://note.com/kirinoha1008/n/n283aa8e4a31b)

アドラー心理学の「課題の分離」は、しばしば「冷たい」「突き放している」と誤解されがちですが、筆者（作業療法士・公認心理師）は、それが「相手を信じる、いちばん温かい行為」であると説いています。
課題の分離の本質は「冷たく突き放す技術」ではなく、「自分がコントロールできない相手の領域（相手の機嫌や反応など）を手放し、自分を許す技術」です。子育てにおいても、「親の思い通りにコントロール（支配）しない」ことで決定権を本人に返しつつ、「求められたら、いつでも援助する」という姿勢を保つことが重要です。線を引くことは分断ではなく、相手の領域を尊重し、相手を信じるからこそできる「愛の線引き」であると解説しています。

コミュニケーションを成果につなげるためには、「自分の話したいこと」ではなく「相手の聞きたいこと」を優先する「相手中心」の切り替えが重要です。相手を観察し、事前にリサーチすることで相手の関心事を見抜き、納得させて行動へ導くプロセスは、AI時代になっても人間にしかできない価値であると強調しています。

## 💻 IT・テクノロジー

### AI時代だからこそ人間は社内ドキュメントをメンテナンスすべき
**Zenn** | [記事リンク](https://zenn.dev/analytics_eng/articles/manage-internal-docs-with-ai)

AIエージェントが開発に参加するようになったことで、社内ドキュメントの「読者」が人間だけでなくAIにも広がりました。これにより、小規模なチームであってもドキュメント化に投資する価値が劇的に高まっています。
しかし、ただAIを情報源に繋ぐだけではノイズが多くなり、コストも跳ね上がります。AIに効率的にドキュメントを活用させるためには、人間が以下の役割を担う必要があります。
1. **AI Slop（低品質な生成コンテンツ）の防止**: AIの出力を鵜呑みにせず、人間が最終的な責任と品質確保を行う。
2. **暗黙知の形式知化**: Agent SkillやADR（Architecture Decision Record）を用いて、AIが理解できる形で知識を体系化する。
3. **CI/CDによるガードレール**: AIがビジネスルールや運用ルールを逸脱しないよう、Policy as Codeなどで自動的にチェックする仕組みを作る。
